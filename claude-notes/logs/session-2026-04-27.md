## Image Block Support

### Add image blocks to story nodes

Added optional `imageUrl` and `imageAlt` properties to the StoryNode schema. When present, the renderer inserts a Block Kit `image` block between the narrative section and the divider.

**Changes:**
- `story/nodes.js` — added `imageUrl` and `imageAlt` to StoryNode typedef; added image to `friday_alert` node
- `game/renderer.js` — `buildStoryBlocks()` and `buildEndingBlocks()` conditionally render image block
- `tests/game/renderer.test.js` — 5 new tests (image rendering, alt text fallback, absence check, ending support)

**Sources:**
- [Block Kit image block](https://api.slack.com/reference/block-kit/blocks#image)

## Error Messages & Start Confirmation

### Add ephemeral error messages and start confirmation to all listeners

Added a shared `postEphemeralError()` helper and wired it into every listener's error path. Previously, errors were logged server-side but users saw silent failures. Now they receive an ephemeral `:warning:` message prompting them to retry.

Also added an ephemeral confirmation when the game is started from a channel context (not a DM), telling the user to check their DMs.

**Changes:**
- `listeners/helpers.js` — added `postEphemeralError(client, userId, text?)` with default message
- `listeners/actions/adventure-choice.js` — ephemeral on unknown node + catch
- `listeners/actions/open-form.js` — ephemeral on invalid node + catch
- `listeners/actions/open-modal.js` — ephemeral on modal open failure
- `listeners/shortcuts/start-adventure.js` — ephemeral channel confirmation + error
- `listeners/views/form-submit.js` — ephemeral on missing state + catch

**Sources:**
- [Slack chat.postEphemeral](https://api.slack.com/methods/chat.postEphemeral)
