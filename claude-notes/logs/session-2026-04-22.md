## Story & Game Logic

### Story graph, game state, and Block Kit renderer

Built the core game engine in three modules:

- `story/nodes.js` — directed graph of 13 nodes with branching choices and 6 endings, each with emoji and summary
- `game/state.js` — per-user in-memory state (current node, choice history, message reference)
- `game/renderer.js` — builds Block Kit blocks for story scenes (header, section, divider, actions, context) and endings (path summary + play again button)

Block Kit elements demonstrated: header, section (mrkdwn), divider, actions (buttons with primary/danger styles), context.

- **Sources:**
  - `story/nodes.js`, `game/state.js`, `game/renderer.js`

## Listeners

### Actions, events, and shortcuts following Bolt patterns

Implemented the full listener suite:

- `listeners/actions/adventure-choice.js` — handles button clicks, advances game state, updates message in place via `chat.update`, handles "Play Again" by posting a new message
- `listeners/events/app-home-opened.js` — publishes App Home tab with game description and "Start Adventure" button
- `listeners/shortcuts/start-adventure.js` — global shortcut and App Home button both start a new game via DM

Each category has an `index.js` for registration, wired through `listeners/index.js`.

- **Sources:**
  - `listeners/actions/`, `listeners/events/`, `listeners/shortcuts/`

## Claude Code Skills

### Six project-scoped skills for streamlined development

Created `.claude/skills/` with six skills:

| Skill | Purpose |
|---|---|
| `/add-scene` | Scaffolds story nodes with correct shape, wires choices, validates graph |
| `/validate-blocks` | Checks Block Kit output against Slack spec constraints |
| `/add-listener` | Generates listener file + index registration + test file |
| `/playtest` | Traces story paths for proofreading narrative flow |
| `/slack-api` | Fetches concise reference from docs.slack.dev |
| `/size-it` | Classifies task size and constrains effort accordingly |

Skills follow official Claude Code best practices (code.claude.com/docs/en/skills): YAML frontmatter, `disable-model-invocation` for action-oriented skills, `allowed-tools` for pre-approved permissions, `$ARGUMENTS` for input.
